# Components and templates *zUIx* components and templates aim to transform and enhance functionality and presentation of the page, or a part of it. The good thing about using components and templates is that even if you already filled in and placed all the contents of your website, you can continue to improve its look, usability and functionality without ever touching the pages where the content is placed. Here follow a few prebuilt examples, but if you are interested in building your own components and templates, see the [ZUIX documentation](https://genielabs.github.io/zuix) for further information about this topic. The `data-ui-options="contentOptions.imageCarousel"` is providing a global JavaScript variable that eventually holds options to apply to the component. In this case the variable is defined in the [./app/main.js](./app/main.js) as follow: ```javascript var contentOptions = { // ... imageCarousel: { on: { 'image:click': function(e, carouselData) { // open the full screen viewer zuix.context('image-browser') // methods of 'media-browser' component .items(carouselData.list) .current(carouselData.current) .open(); } } } // ... } ``` so, it basically indicates what to do when an image is clicked and in this case it will open it full screen by using the *Image Browser* component, which is another component that is loaded in the `index.html` file as follow: ```html
